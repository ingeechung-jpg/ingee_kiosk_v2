키오스크를 만들기로 하고 딱 일주일이 지났다. 간단한 웹디자인이지만 별도의 cms를 쓰지않고 구글드라이브 네트워크를 최대한 사용하며 관리 시스템을 구상하고 만들었다는 나름의 뿌듯함이 있다. 사실 일주일 걸렸지만 수정하고 구조를 다시만드느라 2개의 웹사이트를 만든거나 마찬가지…일주일동안 코덱스를 써보면서 AI가 주는 작업적 환경변화를 어느때 보다 많이 느끼고 있다. 이미지를 다루는 직업으로써 사실 생성형 이미지와 영상이 미치는 산업의 영향도 어마어마하지만 개인적으로 느끼는 체감은 프로그램을 에이아이와 하다보면서 확장되는 다른영역에 대한 이해도 향상이 크다. 간단한 정도의 웹사이트 일은 간간히 받아서 하는 정도의 지식이 있는 디자이너로써 간단한 css나 아준간단한 js 코드등 프론트 단에서의 이해정도만 있다. 그 뒤에 백단에서 돌아가는 사이트 구조와 데이터 호출 최적화 등등등 데이터 다루는 부분은 알아야할것도 너무 많고 디자인과 병행해서 할 일도 아니여서 프로그래머에게 부탁하는 정도로 나 말고도 대부분의 디자이너가 그러지 않을까 생각이든다. 이번에 코덱스를 쓰면서 좋은건 알수 없는 블랙박스 같은 백단의 구조를 배워가는 재미가 있달까? 무언가를 배우는것에 있어 사실 배운다기 보다는 직접해보고 경험해가면서 배우는게 최고인데 비전문가가 배우기 어려운 전문영역의 지식을 에이아이와 뚜닥뚜닥 해가면서 배워가는 경험이 매우 새롭달까? 물론 세부적인 코드야 코덱스가 짜니 결론적으로 알아먹기 힘든건 마찬가지긴 하지만 웹디자인을 할때 신경쓰는 부분이 디자인에서 반응속도와 데이터 구조까지 신경이 쓰인다.

처음 간단한 웹사이트를 만들었을 때는 별도의 CMS를 쓰지않고 구글독스와 시트를 앱스크리트를 이용해서 사이트를 만들었다.

```

mermaid

flowchart LR

    subgraph GDRIVE[구글드라이브]



        subgraph DATA

            CRAWL[구글독스; 게시글]

            GSHEET[구글; 데이터]

        end

        subgraph APP_SCRIPT

            APPSCRIPT[마크다운변환]

            API[데이터구조화]

        end

        CRAWL --> APPSCRIPT

        GSHEET --> API

    end

    subgraph GIT

        HTML[HTML]

    end

    APPSCRIPT <--> HTML

    API <--> HTML

    SLOW[속도 느림]

    GDRIVE --- SLOW --- GIT

```

이렇게 만들고 사용해보니 데이터를 읽어서 html 화면에 뿌려주기위해 API를 불러오니 속도가 느리다. 구글에서 읽어들이는 속도가 느리다보니 페이지를 새로고침 할때 마다 데이터 불러들이는 속도가 길어서 구조를 바꿨다. 앱스크리트에서 데이터를 불러와서 구조를 다 만들고 html에 보내는게 아니라. 깃을 사용해서 데이터만 올리고 깃 내부에 가벼운 데이터만가지고 구조를 빌드하고 웹으로 뿌리는 형식. 이과정에서 구글독스를 마크다운으로 변환하고 이것을 깃에 푸시까지 하는 일련의 구조를 앱스크리트로 구현하니 나름 만족하며 뿌듯해했다.

```

mermaid

flowchart LR

    subgraph GDRIVE["구글드라이브"]

        subgraph DATA

            DOC[구글독스 게시글]

            GDATA[구글 데이터]

        end

        subgraph APP_SCRIPT

            MDPROC[마크다운변환]

            DATAPROC[데이터만 깃에 전송]

        end

        DOC --> MDPROC

        GDATA --> DATAPROC

    end

    subgraph GIT["Git"]

        subgraph HTML

            APP[App]

            MD[.md]

            JSON[.json]

        end

    end

    MDPROC --> MD

    DATAPROC --> JSON

    GDRIVE -- 속도 빠름 --> GIT

```

뿌듯해하며 노트를 하나씩 작성하며 사용을 하고있는데, 이 앱스크립트가 독스파일 내부에 저장되는 형식이라서 새글을 쓰려면 기존의 독스파일을 사본으로 만들고 거기에 글을 써야 깃에 올릴수 있는 구조였다. 처음에는 뭐 이정도 귀찮은건 괜찮치 하고 사용하다가 독스 > 마크다운 변환에 필요한 코드를 수정하다보니…수정하면 매번 모든 독스에 일일히 붙여넣기해야 되는 문제가 발생. 그리고 앱스크리트 내부 코드도 문서를 변환하고 깃에 올리는 여러단계의 코드가 들어가니 점점 구조도 복잡해 지기 시작하고 시간도 오래걸린다.
